Funciones sobre paquetes

Concepto

Establece el alcance de tareas que puede realizarse para un paquete en el sistema.  Las tareas son invocadas desde la Administración de cuentas o bien, con algunas restricciones desde la autogestión de servicios.

Para poder utilizar estas tareas es necesario ubicarse en el entorno de un paquete específico dentro de un cuenta, ya que todas las validaciones y reglas de negocio se definen en función de este, la información actualizada de su situación actual es definida y restringe la ejecución de las mismas.

Administración

Para la ejecución de tareas del paquete elegido, se utiliza un formulario selector desplegado a la derecha del menú de selección del nivel de la estructura de cuentas en el nivel adecuado.

Las tareas disponibles se describen en la siguiente relación, sin embargo por aspectos de seguridad y alcance de los roles o perfiles de usuario algunas de estas podrían no estar disponibles en algunos casos, si esa fuera la situación la tarea no aparecerá del todo dentro de las opciones disponibles.

Datos

Corresponde a una consulta de los datos generales del paquete, heredados de su definición como producto, para lo que se utiliza un formulario de consulta, con los siguientes datos.

  • Nombre de Producto Nombre Comercial oficial de los productos o paquetes de la empresa, identifica de forma única cada producto para efectos de comercialización. Es un valor cuya longitud no puede ser menor a un caracter. La identificación de un producto a partir de su nombre es vital para el proceso atencional. t

  • Tarifa básica Es el precio mínimo y básico para cada ciclo de facturación por parte del cliente adquiriente del producto. Por este precio se incluyen las horas estipuladas en el mismo producto en el campo Horas básicas. Está expresado en la moneda definida por un valor numérico mayor o igual a cero, no acepta valores nulos.

  • Unidades base Unidades incluidas en el precio base del tipo de tarifa, calculado en cada ciclo de facturación, este valor por omisión es cero, y se valida una cantidad mayor o igual a cero. Esta unidad está expresada en la unidad de cobro de la Tarifa.

  • Unidades excedente Es el precio por unidad excedente de las unidades base definidas para un tipo de tarifa. Las unidades están definidas por la Unidad de Cobro de la Tarifa y las unidades no cobrables a destajo son las Unidades Base del Tipo de Tarifa. El precio estipulado está expresado en la moneda de la Tarifa. Valida valores numéricos mayores a cero.

  • Correos Cantidad de cuentas de correo. Número entero mayor o igual a cero que indica cuantas cuentas de correo puede acceder un producto.

  • Cuota de correo Es la cuota máxima límite para e almacenamiento de correos electrónicos expresada en Kilobytes. Es un valor numérico mayor a cero que permite valores nulos.

Cambiar paquete

Permite cambiar el producto o paquete a esta estructura particular, para ello se revisa una restricción definida por producto que indica a cuales productos se permite el cambio a partir del actual, filtrando con ella la lista de selección. Para realizar el cambio se debe elegir el Paquete Es una referencia a la incidencia particular de un producto a partir del nombre de este. deseado de la lista, al elegir se confirman las características del mismo y se permite "verificar" el cambio, invocando un formulario de registro para sincronizar los detalles del cambio, utilizando el siguiente formato:

  • Configuración del paquete actual, establece la estructura de logines del paquete actual, define el paquete y una relación con los logines y los servicios que utiliza

  • Configuración del nuevo paquete, establece la estructura de logines del paquete pretendido, se establece a partir del actual con las siguientes consideraciones:

      • Automáticamente hace equivaler los logines actuales con sus pares, adecuando la información adicional como teléfono de conexión Es el número telefónico desde el cual se realiza la conexión a los sistemas de gestión de red, se reporta por estos para cada evento, obviamente cada evento solo podrá tener un teléfono de referencia. si fuera el caso.

      • Los conflictos, es decir aquellos logines que no pueden ser acomodados automáticamente en el nuevo paquete se presentan con las opciones posibles al usuario, valida y no permite inconsistencias, mostrando estas al final de la forma.

  • Opción de ejecución

Adicionalmente ofrece la consulta de las tareas programadas particularmente para el cambio de paquete, brindando así al usuario la oportunidad de valorar su acción.

Retirar paquete

Realiza el retiro de un paquete y la desactivación de sus logines a una cuenta, utilizando una plantilla de registro con los siguientes campos:

  • Motivo de retiro

  • Devolución de depósito Es un indicador que identificará cuando el evento de retiro de un paquete conlleva la devolución del depósito de garantía asociado.

  • Opción de ejecución

Adicionalmente ofrece la consulta de las tareas programadas particularmente para el retiro de paquete, brindando así al usuario la oportunidad de valorar su acción.
