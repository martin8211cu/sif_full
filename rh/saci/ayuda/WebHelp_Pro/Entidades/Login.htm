Login

Concepto

El login es el nombre único de usuario con que se accede a los servicios brindados a través de un producto de la empresa.  El acceso será la combinación de un usuario y la clave o password correspondiente.

Los logines son únicos para la empresa y deben validarse en ese sentido, sin embargo un login en desuso, después de un plazo parametrizado puede volver a utilizarse, considerando que la acción negará la reactivación del login original.

Registro

El registro de se invoca desde el proceso  logines son una serie de campos de captura con la relación de accesos preconfigurada en el paquete. Cada login corresponderá a la prestación de un servicio particular, combinando los de acceso con cuentas de correo.  Para capturar los logines de cada servicio se utilizará una matriz donde cada fila corresponderá con un servicio a los que se configuran los siguientes campos:

  • Login Es el nombre de login o usuario que se quisiera para acceder un servicio particular. Un mismo login puede tener diferentes servicios asociados aunque solo una incidencia de cada uno. en este caso este nombre debe ser único para cualquier otro login en la empresa. No acepta nulos.

  • (Opcionalmente)Teléfono Es un número telefónico, validado y presentado contra el formato parametrizado para el este. Una entidad puede tener diferentes números telefónicos, el primario se validará, es decir no contendrá valores nulos, los teléfonos adicionales pueden ser nulos., este es un valor cuya ocurrencia depende de la configuración del servicio a través del indicador Teléfono requerido Indicador que establece la movilidad del producto, en caso de marcar el campo, en cada venta del paquete se solicitará un número de teléfono y se verificará el mismo en la conexión. Es un campo booleano (de dos estados) donde la marca significa "SI".
